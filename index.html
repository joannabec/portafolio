Sobre mi

Inicié en el mundo de la programación gracias a Laboratoria, antes de eso egresé de la carrera de diseño industrial y ejercí la profesión, tengo experiencia desarrollando productos: investigar sobre el problema o necesidad de un usuario, detectar la causa y a partir de eso brindar soluciones que se ajusten a la tecnología y recursos disponibles.

Para mí no solo es importarte alcanzar el objetivo si no también disfrutar el proceso para lograrlo, esto fue mi mayor motivación para cambiar de profesión y entrar al mundo de la programación. Como recién estoy iniciando mi prioridad es sentar las bases de programación y pensamiento lógico, en el futuro mi objetivo es tener la experiencia y los conocimientos necesarios para poder resolver problemas complejos mediante el código.

Aparte de todo eso también me interesa desarrollar las habilidades blandas ya que son fundamentales para el crecimiento profesional y personal, es por eso que busco y me gusta recibir feedback para poder mejorar.

Proyectos

Pokepedia

Pokepedia es una página web para jugadores de Pokemon Go y fánaticos de la serie de televisión, se puede encontrar información sobre cada uno de los pokemones, sus evoluciones, ataques, tipo y más!

El proceso del proyecto inició con la investigación del tema con usuarios reales para detectar qué información debía aportar la página, luego se realizaron prototipos de baja y alta fidelidad para obtener feedbak e iterar las propuestas de diseño, finalmente pasamos a la parte divertida, el código.

Tecnologías: vanillaJs, CSS, Jest, Figma, Trello

Md-links

Mdlinks es una librería que lee archivos con formato markdown, detecta los enlaces que contiene, a partir de esto obtener los estatus y realizar estadísticas.

Se puede importar como módulo en un proyecto, como sí también posee una Interfaz de Línea de Comandos.

Burger Queen

Burger queen es una aplicación web para un restaurante de comida rápida y para ser usada en tablets. Sirve como sistema para tomar los pedidos de los clientes y hacer el seguimiento de la orden desde el inicio del proceso hasta la entrega. También posee una sección para el administrador donde se gestiona los usuarios y los productos del menú.

Para este proyecto inicialmente fue necesario hacer el mock de una API, para luego consumir una API real desarrollada por otro equipo de trabajo.

Tecnologías: Reactjs, Sass, React testing library, Mockoon, json-server, node-fetch, Figma, Asana
